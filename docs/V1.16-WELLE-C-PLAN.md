# V1.16 — Welle C Plan: Hosting & Deployment auf Cloudflare

**Status:** Draft, awaiting user approval before implementation.
**Author:** Plan synthesised from parallel `security-reviewer` + `architect` reviews on 2026-05-05.
**Predecessors:** V1.16 Welle A (CSP/SRI/TT shipped), Welle B (CSP-Reporting deklariert via meta-tag, receiver-shape-spec normativ).
**Goal:** Deployment-seitig F-3 (CSP-Reporting effective) schließen + die meta-tag-CSP durch eine HTTP-Header-CSP ergänzen, damit `frame-ancestors 'none'` tatsächlich greift und `report-to` (modernes Reporting API) deklarierbar wird.

## 0. TL;DR — Was sich vom initialen Plan geändert hat

Die Multi-Perspective-Review hat sechs substantielle Änderungen erzwungen:

| # | Initial-Plan | Final-Plan | Quelle | Begründung |
|---|---|---|---|---|
| 1 | CF Pages + Pages Function | **Workers + Static Assets** (one Worker hostet Pages + Receiver) | architect 1.1 | CF konsolidiert seit 2025; Workers + Static Assets ist 2026-first-class. Pages bleibt für Legacy. |
| 2 | `_headers`-File | **Worker-emitted Headers** in `lib/security-headers.ts` | architect 1.3 | Audit-fähig + testbar (Vitest); Migration-Pfad zu anderen Cloud-Providern bleibt offen. |
| 3 | R2 per-request-object | **Workers Analytics Engine (Hot Path)** + **R2 Daily Archive** (Cron Trigger) | architect 1.2 + security H3 | R2-per-Object ist Cost-DoS-Vektor (Class-A-Op-Amplification); AE writeDataPoint ist ~free + queryable. R2 wird zur Cold-Audit-Schicht. |
| 4 | `master`-auto-deploy | **PR-Preview + Manual-Promote-via-Tag** | architect 1.4 | Auto-Deploy bricht den Atlas-Workflow (review-gates required); Tag-triggered Production-Deploy passt zur V-Welle-Disziplin. |
| 5 | COOP/COEP weggelassen | **COOP + COEP aktiviert** | security M5 | Spectre cross-process risk auf WASM-cryptographic-verifier ist nicht ignorierbar; COEP `require-corp` bricht nichts (alle Assets same-origin). |
| 6 | 90 Tage Retention | **30 Tage Retention + Daily-Summary 1 Jahr** | security M1 | 90-Tage-Per-Object × Attack-Rate ist unbounded Cost-Surface. 30-Tage-Raw + 1-Jahr-aggregierte-Summaries ist Operations-Best-Practice. |

Nicht-trivialer **dropped concern** (architect 2.1 — Sub-Subdomain `csp.atlas-trust.dev` für Quota-Isolation): Bewusst NICHT übernommen für V1.16 Welle C. Begründung: Single-Worker ist im Free-Tier komfortabel (100k req/day); Sub-Subdomain-Split kostet Komplexität ohne unmittelbaren Gewinn. Als **Welle D Candidate** dokumentiert für den Fall, dass Receiver-Quota tatsächlich limitierend wird.

### 1.3 Final Header-Set (Worker-emitted, jeder static-asset Response)

```ts
{
  // CSP — same policy as page-bytes-meta-tag, plus report-to/Reporting-Endpoints
  // (the meta-tag in index.html stays as defence-in-depth for non-Worker hosts)
  "Content-Security-Policy":
    "default-src 'none'; " +
    "script-src 'self' 'wasm-unsafe-eval'; " +
    "style-src 'self' 'unsafe-inline'; " +
    "connect-src 'self'; " +
    "form-action 'none'; " +
    "frame-ancestors 'none'; " +
    "base-uri 'none'; " +
    "require-trusted-types-for 'script'; " +
    "trusted-types 'none'; " +
    "report-uri /csp-report; " +
    "report-to csp-endpoint",

  // Modern Reporting API endpoint group declaration
  "Reporting-Endpoints": 'csp-endpoint="/csp-report"',
  // Legacy Report-To (some browser versions still honour the older syntax)
  "Report-To":
    '{"group":"csp-endpoint","max_age":10886400,"endpoints":[{"url":"/csp-report"}]}',

  // HSTS — long max-age, includeSubDomains, preload directive (NOT submitted yet)
  "Strict-Transport-Security": "max-age=63072000; includeSubDomains; preload",

  // Misc hardening
  "X-Content-Type-Options": "nosniff",
  "Referrer-Policy": "no-referrer",

  // Cross-origin isolation (security M5 — closes Spectre cross-process risk)
  "Cross-Origin-Opener-Policy": "same-origin",
  "Cross-Origin-Embedder-Policy": "require-corp",
  "Cross-Origin-Resource-Policy": "same-origin",

  // Permissions-Policy — disable every browser feature we don't use
  "Permissions-Policy":
    "accelerometer=(), camera=(), geolocation=(), gyroscope=(), " +
    "magnetometer=(), microphone=(), payment=(), usb=(), midi=(), " +
    "publickey-credentials-get=()",
}
```

### 1.4 Cache-Control per Pfad

| Pfad | Cache-Control | Begründung |
|---|---|---|
| `/index.html` | `no-cache, must-revalidate` | Hat SRI-Pin auf `app.js` — wenn `app.js`-Hash sich ändert, muss `index.html` neu gefetcht werden |
| `/app.js` | `public, max-age=31536000, immutable` | Hash-pinned via SRI — Bytes ändern sich = neuer Filename oder neuer Hash → immutable safe |
| `/pkg/*` | `public, max-age=31536000, immutable` | Wasm-bindgen output, content-hashed filenames |
| `/csp-report` | `no-store` | Receiver-Response, soll niemals cached werden |

## 2. Receiver-Implementation Spec (post-review hardened)

### 2.1 Validation-Pipeline (in dieser Reihenfolge — silent-204 bei jedem Step)

```
1. Method-Check: nur POST, sonst 405 (kein silent-204 hier — clients sollen wissen, dass sie das falsche Verb nutzen)
2. Origin-Check (CSRF-Anker):
   - request.headers.get('Origin')
   - reject wenn null/missing/'null' (security H1)
   - reject wenn !== 'https://playground.atlas-trust.dev'
3. Content-Type-Allowlist:
   - 'application/csp-report' (legacy report-uri)
   - 'application/json' (modern Reporting API)
   - reject sonst (silent 204)
4. Body-Cap pre-read:
   - Content-Length-Header > 65536 → silent 204
5. Body read + post-read cap:
   - body.length > 65536 → silent 204
6. Depth-limited JSON-Parser (security H2):
   - max 10 levels nesting
   - max 100 keys per object
   - reject deeply-nested (silent 204)
7. Schema-Validation:
   - top-level 'csp-report' OR 'body' field present
   - reject sonst (silent 204)
8. ANSI-Strip rekursiv:
   - alle String-Felder durch /[\x1B\x9B][[\]()#;?]*([\d]{1,4}(;[\d]{0,4})*)?[\d\w]/g
9. Normalize:
   - extract specific allowlisted fields, drop everything else
   - { violated_directive, blocked_uri (URL-parsed only), document_uri (origin only), source_file (basename only), line_number (int), column_number (int), original_policy (truncated to 1024 chars) }
10. Rate-Limit-Check (post-validation, pre-write):
    - per /64-IPv6-prefix or /32-IPv4 → 100/min
    - global cap 1000/min via Durable Object counter
    - exceeded → silent 204 (nicht persistiert, nicht logged-as-error)
11. AE Write:
    - env.CSP_REPORTS_AE.writeDataPoint({ blobs: [normalized fields...], doubles: [line, column], indexes: [violated_directive] })
12. Response:
    - new Response(null, { status: 204 })  // empty body, NO custom headers (sec L2 Vary-injection-mitigation: explicitly NO Vary)
```

### 2.2 Internal Operator Logging (security M2)

Auf jedem Validation-Failure: `console.error(JSON.stringify({ category, request_id, cf_ray, ... }))`.
Nicht im 204-Response sichtbar; nur via `wrangler tail` oder CF Dashboard Workers-Logs.

Categories:
- `origin_missing`, `origin_null`, `origin_mismatch`
- `content_type_rejected`
- `body_too_large_pre`, `body_too_large_post`
- `json_parse_error`, `json_too_deep`, `json_too_wide`
- `schema_invalid`
- `rate_limited_per_ip`, `rate_limited_global`
- `ae_write_failed`

### 2.3 Daily Archive (Cron Trigger)

`@ 03:00 UTC daily`: Worker-Cron-Handler queryt AE für vergangenen Tag, formatiert JSONL, R2-PutObject-once (`archive/YYYY-MM-DD.jsonl`). R2 Lifecycle-Rule: 365 Tage Retention (1 Jahr Audit-Trail).
