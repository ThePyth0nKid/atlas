#### V1.11 Scope A wave-3 — sealed per-workspace signer (commits `1c0e288`, `887b3c6`, `0454a85`, `c289be5`)

**The trust property:** when wave-3 is opted in via `ATLAS_HSM_WORKSPACE_SIGNER=1` AND the HSM trio is set, the per-tenant Ed25519 signing scalar **never enters Atlas address space, even transiently**. Per-workspace keypairs are generated via `CKM_EC_EDWARDS_KEY_PAIR_GEN` with attributes `Sensitive=true, Extractable=false, Derive=false`; signing routes through `CKM_EDDSA(Ed25519)`; only the 64-byte signature exits the device. Closes the wave-2 residual ("per-tenant scalar transits a `Zeroizing<[u8; 32]>` for the lifetime of one `sign` call").

**The `WorkspaceSigner` trait** (`crates/atlas-signer/src/workspace_signer.rs`):

```rust
pub trait WorkspaceSigner: Send + Sync {
    fn sign(&self, workspace_id: &str, signing_input: &[u8]) -> Result<[u8; 64], WorkspaceSignerError>;
    fn pubkey(&self, workspace_id: &str) -> Result<[u8; 32], WorkspaceSignerError>;
}
```

Two implementations:
- `DevWorkspaceSigner` — wraps `Box<dyn MasterSeedHkdf>`, delegates to `derive_workspace_signing_key_via`. The V1.9/V1.10 path expressed in wave-3 terminology. Produces byte-identical signatures to V1.10 (locked by `phase-a determinism witness` tests).
- `Pkcs11WorkspaceSigner` (`crates/atlas-signer/src/hsm/pkcs11_workspace.rs`) — sealed signer. Per-workspace keypairs persisted under canonical label `atlas-workspace-key-v1:<workspace_id>` with `CKA_ID = CKA_LABEL` (PKCS#11 §10.1.2 binding).

**The dispatcher** (`workspace_signer_loader_with_writer` in `workspace_signer.rs`):

| Layer | Activation | Per-tenant key location |
|-------|-----------|------------------------|
| 1. Wave-3 sealed | `ATLAS_HSM_WORKSPACE_SIGNER=1` truthy AND HSM trio set | inside HSM (CKM_EDDSA) |
| 2. Wave-2 sealed-seed | wave-3 unset AND HSM trio set | derived in-process (HKDF), master seed inside HSM |
| 3. Dev | wave-3 unset AND HSM trio unset | derived in-process from `DEV_MASTER_SEED` |

Fail-closed at every layer: wave-3 opted in with no trio → refuses; partial trio under wave-3 opt-in → refuses; module-open failure → refuses. No silent fallthrough.

**Production CLI surface:**
- `atlas-signer derive-pubkey --workspace <ws>` — works under all three layers; under wave-3, generates the keypair on first call and finds-it-by-label on subsequent calls (lazy provision).
- `atlas-signer sign --derive-from-workspace <ws> ...` — routes through the dispatcher; under wave-3, `CKM_EDDSA` runs inside the HSM.
- `atlas-signer derive-key --workspace <ws>` — **structurally refused under wave-3** (`run_derive_key_or_refuse` in `main.rs` reads `ATLAS_HSM_WORKSPACE_SIGNER` directly and exits with code 2). The sealed scalar is unexportable; refusing at the CLI surface preserves the V1.10 fail-closed pattern.

**The wave-3 truth-table audit signal extends V1.10's:**
- HSM trio set + `ATLAS_HSM_WORKSPACE_SIGNER=1` + binary built with `--features hsm` → wave-3 production
- HSM trio set + `ATLAS_HSM_WORKSPACE_SIGNER` unset → wave-2 sealed-seed
- HSM trio unset + `ATLAS_DEV_MASTER_SEED=1` → dev seed (CI / dev hosts only)
- Anything else → fail-closed at the gate

14. **PKCS#11 is verbose; cryptoki abstracts the syntax but not the semantics.** Test against SoftHSM2 with pinned-pubkey goldens before trusting any vendor's HKDF/EDDSA. (V1.10)
15. **Drop order matters for FFI resources.** `Mutex<Session>` BEFORE `_ctx: Pkcs11` so `C_CloseSession` runs before `C_Finalize`. (V1.10)
16. **Absolute-path guard at config-parse time, not at use time.** Platform-conditional test data on Windows vs Unix. (V1.10)
17. **Zeroize at every heap-resident copy of secret material.** PIN bytes, HKDF output, per-tenant seed array — all wrapped. (V1.10)
18. **Operator runbook ceremonies must use secrets-manager env vars, not literal PINs.** `umask 077`, source from secrets manager, `unset` at end. **wave-3 ceremony (V1.11) further upgrades this to `--pin-source env:VAR` for `pkcs11-tool` so even argv exposure is closed.** (V1.10/V1.11)

### NEW for V1.12 (V1.11 lessons)

19. **Pre-flight test work pays for itself.** The `hsm_byte_equivalence` integration test (commit `4e63963`) was instrumental in committing to wave-3's "explicit opt-in because pubkeys rotate" design vs. the alternative "implicit upgrade because pubkeys are byte-equivalent". Without that test, the design might have been wrong; with it, the rotation event is documented + tested + understood-by-operators. **Apply the same pattern to V1.12 scope decisions where vendor behaviour is uncertain.**

20. **Trait dyn-safety is non-trivial and worth pinning explicitly.** `WorkspaceSigner` is `Send + Sync` + dyn-safe (no generics, no `Self` returns, no `async fn`) — this is what lets the dispatcher return `Box<dyn WorkspaceSigner>` and pick the impl at runtime. Document this constraint in the trait doc-comment so future contributors don't reach for `async fn` and break the dispatch shape.

21. **Layered fail-closed dispatchers need explicit refusal text per layer.** `workspace_signer_loader_with_writer` has three refusal sites (wave-3 opt-in without trio, partial trio, module-open failure), each with its own actionable message. Operators reading the error get a one-line remediation hint that names the env var to fix. This pattern (a) prevents silent fallthrough to weaker layers and (b) gives ops a fast diagnostic path. Apply to any future layered dispatcher.

22. **`derive-key` style "structurally refused" subcommands are a real defensive pattern.** Under wave-3, `derive-key` exits with code 2 because the sealed scalar is unexportable. The refusal fires from a direct env-var read (not via the dispatcher) so it triggers even when the HSM trio is incomplete — refusing at the CLI surface is cheaper and more honest than letting the operator hit a deeper unexportability error. **Use this pattern when adding subcommands that conceptually contradict a deployment mode.**

23. **PIN-on-argv is unavoidable on some legacy CLIs (softhsm2-util's init-token).** When the CLI surface forces argv exposure, document it honestly in the runbook + recommend host-hardening (single-tenant ceremony host, `/proc` `hidepid=2`, bounded one-shot exposure). Don't paper over with "the value is in env, not history" framing — that conflates shell history with `/proc/<pid>/cmdline` and trains operators to assume false safety.

24. **Section renumbering in long runbooks is reversible if anchors are textual, not numeric.** `workspace_signer.rs:503` references `§wave-3` (literal text) rather than `§3`, so the V1.11 doc-sync's renumber (§3-§7 → §4-§8) didn't break the reference. Section numbers shift; section names usually don't. **Prefer textual anchors over numeric in cross-reference comments.**
