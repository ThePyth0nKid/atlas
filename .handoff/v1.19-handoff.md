## 3. V1.19 Welle 1 ‚Äî implementation context (rationale, threat model, hardenings)

**State summary:** first user-facing write path. After 18 waves of verify-side and distribution-side hardening (V1.0 ‚Üí V1.18), V1.19 pivots to product surface. A Browser-User can now create an Atlas node via `POST /api/atlas/write-node` + `/write` page (Next.js 16 App Router, React 19, TS-strict). Trust-Inkrement structurally identical to MCP-host writes ‚Äî same `atlas-signer` Rust binary, same HKDF-SHA256 workspace-key derivation, same append-only `events.jsonl` format ‚Üí the Verifier cannot distinguish web-emitted from MCP-emitted events because they're byte-identical at the AtlasEvent envelope layer.

**Threat model & deliberate non-goals (from `route.ts:8-51` source comment):**

1. **NO authentication.** Atlas's trust-model is key-based (every event records `signature.kid`), not user-based. Deployer gates exposure at network/proxy layer ‚Äî see `OPERATOR-RUNBOOK ¬ß17`. Surfacing this route on a public address without an upstream auth gate IS the operator's choice and responsibility. Adding half-baked auth would be worse than none ‚Äî operators would assume security exists when it's only a thin wrapper.

2. **Per-tenant kid auto-derived from `workspace_id`.** Route NEVER accepts a caller-supplied `kid`. Eliminates "sign as somebody else" affordance the MCP tool intentionally reserves for trusted MCP hosts (Cursor, Claude Desktop). Deterministic via `perTenantKidFor(workspace_id) ‚Üí "atlas-anchor:{workspace_id}"`.

3. **Single-process locking only.** Per-workspace mutex in `lib/atlas/event.ts` serialises writes within ONE Node process. Running atlas-web AND atlas-mcp-server against the same workspace concurrently CAN fork the DAG. Verifier accepts forks (it's a DAG), but the user's mental model breaks. Deploy one writer per workspace until V2 ships an external lock.

4. **`ATLAS_DEV_MASTER_SEED=1` must be set** in the Next.js process environment (or the wave-3 HSM opt-in), otherwise per-tenant subcommands of `atlas-signer` refuse to start. Surfaced as 500 with a clear message when missing.

**Web-specific hardenings (vs MCP variant):**

- **Body-size + attributes cap (defence-in-depth):** `REQUEST_BODY_MAX_BYTES = 256 KB` (raw HTTP body, Content-Length pre-check before `req.json()` reads stream) + `ATTRIBUTES_MAX_BYTES = 64 KB` (Zod `.refine()` on JSON-serialised attributes). 64 KB is well within Linux ARG_MAX (~2 MB) ‚Äî payload passes to `atlas-signer` as `--payload <json>` argv string, so 1 MB attributes would E2BIG ‚Üí generic 500. 64 KB gives clean Zod error at boundary instead.
- **Signer-subprocess timeout + stdout cap:** `SIGN_TIMEOUT_MS = 30_000`, `DERIVE_TIMEOUT_MS = 5_000`, `STDOUT_CAP_BYTES = 512 KB`. Settle-pattern with AbortController-style guard prevents double-resolve on timeout-vs-close race. clearTimeout in both close + error handler.
- **`redactPaths` on 500-responses:** strips absolute filesystem paths from error messages ‚Äî defends against path-disclosure via 500. Applied to `SignerError.message` and `unexpected: ${err.message}` paths. Defence-in-depth: signer should not write paths to stderr in the first place, but redacting here means we don't depend on that discipline. `StorageError` messages already path-sanitised at source (`storage.ts:sanitiseFsError`).
- **Per-workspace mutex Map-identity-race-closed (CRITICAL pre-resume fix):** `if (workspaceMutex.get(workspaceId) === next) workspaceMutex.delete(workspaceId)` ‚Äî identity comparison before deletion closes race where concurrent caller already set'd a newer tail and this callback would otherwise delete the newer tail from the map. **Applied to BOTH `apps/atlas-web/src/lib/atlas/event.ts` AND `apps/atlas-mcp-server/src/lib/event.ts`** for parity (and as preparation for V1.19 Welle 2 bridge-extraction).
- **Spread `attributes` FIRST, schema-checked fields win:** `{ ...args.attributes, kind: args.kind, id: args.id }` ‚Äî never let a caller-supplied attribute object overwrite the schema-checked node identity. Silent override would corrupt the signed payload.
- **`existsSync` guard on `ATLAS_SIGNER_PATH` cache:** if env-var path doesn't exist on disk, fall through to workspace candidates instead of caching an invalid path that would ENOENT-fail every subsequent sign.
