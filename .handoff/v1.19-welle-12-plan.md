## Decisions (final)

- **Negative CLI test scope:** ONE positive happy-path test (`strict_chain_passes_linear_bank_trace`). The 9 hashchain unit tests in `crates/atlas-trust-core/src/hashchain.rs` already cover empty-trace / two-genesis / zero-genesis / sibling-fork / DAG-merge / self-reference at the library surface. Adding a forked-DAG fixture to the CLI test would require pre-signed event-hashes (or a sign-with-mutation harness) for marginal additional coverage — not load-bearing.

- **SemVer audit format:** standalone doc at `docs/SEMVER-AUDIT-V1.0.md`. Cleaner than embedding in the plan-doc; survives post-merge as the v1.0 baseline.

- **CHANGELOG format:** [Keep-a-Changelog](https://keepachangelog.com/en/1.1.0/) style, hand-authored from the Welle 1–11 SHIPPED blocks in `.handoff/v1.19-handoff.md`. Single root-level `CHANGELOG.md` consolidating all crates + packages (Atlas ships as a coherent system, not as independent libraries).

- **Sub-path export gating for `__*ForTest` family:** decide based on SemVer audit findings. If the four exports (`__redactPathConstantsForTest`, `__signerLimitsForTest`, `__signerBinaryCacheForTest`, `__REQUEST_BODY_MAX_BYTES_FOR_TEST`) materially expand the v1.0 public surface, gate them in Welle 12. If they're "test ergonomics, public-but-clearly-internal", defer to post-1.0 minor bump per documented `__` prefix convention.

- **`--require-strict-chain` default remains opt-in.** Per Welle 9 trust-note: Atlas is fundamentally a DAG; forks are valid wire shape under multi-process-writer deployments. Strict mode is the auditor-facing gate. The SemVer audit will document this as an intentional v1.0 contract.

- **atlas-web e2e step-3 negative path (malformed input) is NOT augmented.** The `--require-strict-chain` flag is a verifier-time enforcement; the route-handler 400 path doesn't reach the verifier. Mirroring smoke.ts step-8 exclusion rationale.

## Test cases (concrete)

**`strict_chain_passes_linear_bank_trace`** in `crates/atlas-verify-cli/tests/strict_mode.rs`:
```rust
#[test]
fn strict_chain_passes_linear_bank_trace() {
    // The bank-demo trace is a 5-event linear chain (1 genesis +
    // 4 single-parent successors, no forks). `--require-strict-chain`
    // must accept it with exit 0 and surface the strict-chain evidence
    // row + Strict flags line.
    let output = Command::new(cli_bin())
        .args([
            "verify-trace",
            bank_trace().to_str().expect("trace path is utf8"),
            "--pubkey-bundle",
            bank_bundle().to_str().expect("bundle path is utf8"),
            "--require-strict-chain",
        ])
        .output()
        .expect("failed to spawn atlas-verify-cli");

    let stdout = String::from_utf8_lossy(&output.stdout);
    assert!(
        output.status.success(),
        "strict-chain verify must exit 0 on linear bank trace.\nstdout:\n{stdout}",
    );
    assert!(stdout.contains("VALID"));
    assert!(stdout.contains("strict mode"));
    assert!(
        stdout.contains("require_strict_chain"),
        "strict-chain run must list the flag in 'Strict flags:'; got:\n{stdout}",
    );
    assert!(
        stdout.contains("strict-chain"),
        "evidence list must contain the strict-chain check line; got:\n{stdout}",
    );
    assert!(
        stdout.contains("form a strict linear chain"),
        "evidence row must contain happy-path prose; got:\n{stdout}",
    );
}
```

Negative path coverage already exists at the hashchain.rs unit-test layer (9 tests). The CLI integration surface adds nothing because the bank trace is linear — synthesising a forked-DAG fixture would require pre-signed event hashes.

**`apps/atlas-web/scripts/e2e-write-roundtrip.ts` step-6 augmentation:**
- Append `--require-strict-chain` to the `spawnSync` argv (currently has `--require-per-tenant-keys` only). The 2-event linear write produces a clean strict-chain pass.
- Add two regex assertions on stdout (mirror smoke.ts Welle 10 pattern, anchored to `Strict flags:` prefix to prevent vacuous pass on stray identifier echo):
  - `/✓ strict-chain — \d+ event\(s\) form a strict linear chain/` — evidence-row pin (check label + count + happy-path prose + ok-marker `✓`)
  - `/Strict flags:[^\n]*require_strict_chain/` — flag-name pin anchored to the `Strict flags:` line prefix
- Rationale comment block explaining: (a) the 2-event WORKSPACE leg is structurally guaranteed linear by the per-workspace mutex serialisation in `writeSignedEvent`; (b) any future regression producing a fork-DAG (e.g. multi-process writer, broken mutex) turns this gate red; (c) the assertions mirror the three-lane MCP-server smoke pattern from Welle 10.

## SemVer audit scope (concrete checklist for `docs/SEMVER-AUDIT-V1.0.md`)

The audit doc surveys five public-API surface categories. Each item gets one of four risk-tags: **Locked** (committed for v1.0; breaking change → SemVer-major), **Locked-Behind-Flag** (opt-in surface, default stable), **Internal-but-Exported** (`__*ForTest` family — `__` prefix as documented "not a stability contract" boundary), **Defer-Decision** (post-1.0 minor candidate).

1. **Rust public types** (`crates/atlas-trust-core` re-exports + per-crate exports):
   - `TrustError` (`#[non_exhaustive]` ✓) — new variants are SemVer-minor. **Locked.**
   - `VerifyOptions` struct — adding new `require_*` fields with `Default::default()` semantics is SemVer-minor. **Locked.**
   - `VerifyOutcome` struct — adding new evidence-list entries or summary fields is SemVer-minor. **Locked.**
   - `AtlasEvent`, `AtlasTrace`, `PubkeyBundle`, `EventSignature` — wire-format types. **Locked**, schema-version-gated.
   - `check_strict_chain`, `compute_tips`, etc. — free functions. **Locked.**
   - `WitnessSig`, `witness_signing_input` — V1.13 witness surface. **Locked.**

2. **Rust CLI flags** (`atlas-verify-cli`):
   - `verify-trace --require-per-tenant-keys`, `--require-anchors`, `--require-anchor-chain`, `--require-witness N`, `--require-strict-chain`, `-o json|human`. Each flag's effect is part of the v1.0 contract.
   - Adding new flags = SemVer-minor. Removing or changing flag semantics = SemVer-major. **Locked.**
   - `-o json` output schema = `VerifyOutcome` serde — see #1.

3. **HTTP wire shapes** (`apps/atlas-web/src/app/api/`):
   - `POST /api/atlas/write-node` request body Zod schema + 200 response shape + 400/413 error shapes. **Locked.**
   - `GET /api/atlas/write-node?workspace_id=…` → `{ kid }`. **Locked.**
   - `/api/golden/*` static export shape. **Locked.**
   - `REQUEST_BODY_MAX_BYTES = 256 KB` cap — operator-tunable in future, but the cap-and-413 contract is **Locked**.

4. **npm package exports:**
   - `@atlas-trust/verify-wasm` — `VerifierHandle`, `verify_trace_json`, `verifier_version`. **Locked.**
   - `@atlas/bridge` — re-exports listed below. The four `__*ForTest` exports get **Internal-but-Exported** unless sub-path gating happens in Welle 12.
   - `@atlas/bridge` public re-exports (counted from `packages/atlas-bridge/src/index.ts`):
     - Types: `AtlasEventValidated`, `PubkeyBundle`, `AtlasTrace`, `AtlasEvent`, `EventSignature`, `SchemaVersion`, `WriteEventArgs`, `WriteEventResult`, `UlidState`, `RandomSource`
     - Functions: `writeSignedEvent`, `appendEvent`, `readAllEvents`, `signEvent`, `redactPaths`, `stringifyAnchorJson`, `resolveSignerBinary`, `repoRoot`, `workspaceDir`, `perTenantKidFor`, `TEST_IDENTITIES`, `ulid`, `createUlid`, `nextUlid`
     - Internal exports (4): `__signerBinaryCacheForTest`, `__signerLimitsForTest`, `__redactPathConstantsForTest`, plus `__REQUEST_BODY_MAX_BYTES_FOR_TEST` from `apps/atlas-web/src/app/api/atlas/write-node/route.ts`

5. **MCP tool surface** (`apps/atlas-mcp-server/src/tools/`):
   - Tool names + arg schemas + result shapes. MCP SDK convention is name-based discovery; renaming a tool = SemVer-major. Adding tools = SemVer-minor. **Locked.**

**Audit deliverable per item:** risk-tag, short rationale, optional pre-v1.0 hardening recommendation. Total expected length: ~150-250 lines.

**Sub-path export gating decision rule:** if the four `__*ForTest` exports are all reachable from a single sub-path export `@atlas/bridge/test-internals` without consumer breakage (atlas-mcp-server tests are the only consumers; route.ts test would move to `apps/atlas-web/src/app/api/atlas/write-node/__test-internals__.ts` or similar), gate them in Welle 12 and bump `@atlas/bridge` to a coordinated SemVer-minor in the same commit. Otherwise defer to post-1.0.
